STM32 RTC. Прерывания

Существует три события, которые могут генерировать прерывание от RTC:

  • переполнение входного делителя (т.е. секундные импульсы);
  • переполнение счетного регистра;
  • совпадение сигнального и счетного регистров.

Чтобы использовть прерывания от RTC необходимо выполнить три шага.

 

Шаг 1. Создать функцию-обработчик прерывания.   Она будет иметь следующий вид:

//********************************************************************************************
//function  прерывание, генерируемое RTC                                                    //
//********************************************************************************************
void RTC_IRQHandler(void)
{
  //причина прерывания - переполнение входного делителя (новая секунда)
  if(RTC->CRL & RTC_CRL_SECF)
  {
     RTC->CRL &= ~RTC_CRL_SECF;    //сбросить флаг (обязательно!!!)
     //выполняем какие-то действия
  }
 
  //причина прерывания - совпадение счетного и сигнального регистра
  if(RTC->CRL & RTC_CRL_ALRF)
  {
     RTC->CRL &= ~RTC_CRL_ALRF;    //сбросить флаг (обязательно!!!)
     //выполняем какие-то действия
  }
 
  //причина прерывания - переполнение счетного регистра
  if(RTC->CRL & RTC_CRL_OWF)
  {
     RTC->CRL &= ~RTC_CRL_OWF;     //сбросить флаг (обязательно!!!)
     //выполняем какие-то действия
  }
 
}

Как видим, функция-обработчик одна, а причину, вызвавшую вызов этой функции, можно определить проверкой флагов соответствующих событий. 

Нашли причину прерывания, сбросили флаг и выполняем какие-то действия.

Сбрасываем флаг обязательно. Иначе событие будет постоянно генерировать прерывание.

 

Шаг 2. Активировать событие от RTC, которое будет вызывать прерывание:

RTC->CRL |=  RTC_CRL_CNF;  //разрешить конфигурирование регистров RTC
RTC->CRH  =  RTC_CRH_SECIE;//разрешить прерывание от секундных импульсов
RTC->CRH  =  RTC_CRH_ALRIE;//разрешить прерывание при совпадении счетного и сигнального регистра
RTC->CRH  =  RTC_CRH_OWIE; //разрешить прерывание при переполнении счетного регистра
RTC->CRL &= ~RTC_CRL_CNF;  //выйти из режима конфигурирования

 

Шаг 3. Разрешить прерывание от модуля RTC:

NVIC_EnableIRQ (RTC_IRQn);      //вызвать функцию, которая разрешит прерывание от модуля RTC

 

Небольшое, но важное уточнение.
В статье «STM32 RTC. Инициализация и использование» приводится пример функции инициализации часов.
В ней есть такая операция: «PWR->CR |= PWR_CR_DBP».
Благодаря этому действию разрешается доступ к области резервных данных для записи. Не выполнив этой операции невозможно производить запись, поэтому используется это действие.
Но для обеспечения защиты данных в области BKP, необходимо сразу после изменения данных запретить запись в область BKP, выполнив операцию «PWR->CR &= ~PWR_CR_DBP»(т.е. сбросить флаг DBP регистра PWR->CR).
Соответственно, в обработчике прерывания от RTC, чтобы сбросить какой-то флаг необходимо: разрешить запись в область BKP, сбросить флаг, запретить запись в область BKP.
Активировать событие, вызывающее прерывание, необходимо по такой же схеме.

    uint8_t SecondCounter = 0;      //счетчик 10 секунд
void RTC_IRQHandler(void)
{
  //причина прерывания - переполнение входного делителя (новая секунда)
  if(RTC->CRL & RTC_CRL_SECF)
  {
     RTC->CRL &= ~RTC_CRL_SECF;    //сбросить флаг (обязательно!!!)
     SecondCounter++;              //увеличить счетчик секунд
     if(SecondCounter==10)         //прошло 10 секунд, выполняем нужную работу
     {
      SecondCounter=0;             //очистить счетчик для следующего цыкла
      //.... выполняем что-то нужное
     }     
 
}

    Второй вариант — использовать сигнальный регистр.
    При инициализации загружаем в него значение на 10 больше значения счетного:

    uint32_t tmp;
RTC->CRL |= RTC_CRL_CNF; //разрешить конфигурирование регистров RTC
tmp = (uint32_t)((RTC->CNTH << 16) | RTC->CNTL);
tmp+=10;
RTC->ALRH = tmp>>16;
RTC->ALRL = tmp;
RTC->CRH  =  RTC_CRH_ALRIE;         //разрешить прерывание от сигнального регистра
RTC->CRL &= ~RTC_CRL_CNF;           //выйти из режима конфигурирования

    а затем в прерывании от сигнального регистра увеличивать его постоянно на 10

    Третий вариант — использовать таймер общего назначения, настроив его соответственно.
