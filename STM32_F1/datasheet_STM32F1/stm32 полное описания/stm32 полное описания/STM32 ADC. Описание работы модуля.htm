STM32 ADC. Описание работы модуля

Продолжаю исследование теории по АЦП микроконтроллеров STM32. Перевел некоторые части документации с “ангельского” на русский. Решил опубликовать этот перевод со своими комментариями. Может кому пригодится.

Режим одиночного преобразования.

В этом режиме АЦП выполняет всего одно преобразование. Он стартует после установки разряда ADON регистра ADC_CR2 (только для регулярных каналов) или от внешнего сигнала (для регулярного и инжектированного каналов). При этом разряд CONT регистра ADC_CR2 должен быть равен нулю.

После окончания преобразования в выбранном канале выполняется:

  • для регулярного канала:
    • Результат преобразования сохраняется в регистре ADC_DR
    • Устанавливается флаг EOC
    • Генерируется прерывание, если установлен разряд EOCIE
  • для инжектированного канала:
    • Результат преобразования сохраняется в регистре ADC_DRJ1
    • Устанавливается флаг JEOC
    • Генерируется прерывание, если установлен разряд JEOCIE

После этого АЦП останавливается

Режим непрерывного преобразования.

В этом режиме АЦП начинает следующее преобразование как только закончится первое.

Этот режим стартует от внешнего источника или при установке разряда ADON регистра ADC_CR2, при этом разряд CONT регистра ADC_CR2 должен быть равен единице.

После каждого преобразования выполняется:

  • для регулярного канала:
    • Результат преобразования сохраняется в регистре ADC_DR
    • Устанавливается флаг EOC
    • Генерируется прерывание, если установлен разряд EOCIE
  • для инжектированного канала:
    • Результат преобразования сохраняется в регистре ADC_DRJ1
    • Устанавливается флаг JEOC
    • Генерируется прерывание, если установлен разряд JEOCIE

Режим сканирования.

Этот режим используется для сканирования группы каналов.

Он выбирается установкой разряда SCAN регистра ADC_CR1. Если этот разряд установлен, АЦП сканирует все каналы, выбранные в регистрах ADC_SQRx (для регулярных каналов) или регистре ADC_JSQR (для инжектированных каналов). Для каждого канала группы выполняется одно преобразование. После окончания преобразования следующий канал (его преобразование) запускается автоматически. Если разряд CONT установлен, то преобразование не останавливается на последнем канале, а вновь запускается с первого канала.

Если установлен разряд DMA, контроллер прямого доступа к памяти используется для передачи результата в память после окончания каждого преобразования (после установки разряда EOC).

В инжектированных каналах результат сохраняется в регистрах ADC_JDRx.

Analog watchdog (он же оконный компаратор).

Флаг AWD устанавливается, если напряжение измеренное АЦП, больше или меньше заданного уровня. Этот уровень задается регистрами ADC_HTR и ADC_LTR. Прерывание может быть разрешено разрядом AWDIE регистра ADC_CR1.

Как видно из этой диаграммы, измеряемое напряжение должно находится в области между HTR и LTR.

При его выходе из указанной области будет установлен флаг AWD со всеми вытекающими последствиями.

Разряд AWDSGL задает количество контролируемых каналов. Если он равен 0 – все каналы, 1 – один канал (номер канала при этом задается в разрядах AWDCH[4:0]).

Разряд AWDEN разрешает контроль каналов регулярной группы, а JAWDEN – контроль каналов инжектированной группы.

 

Управление инжектированными каналами.

 Запускаемое инжектирование.

Чтобы использовать этот режим (режим, когда каждый цикл необходимо запускать) необходимо чтобы был сброшен разряд  JAUTO и установлен разряд SCAN регистра ADC_CR1.

  1. Запуск преобразования группы инжектированных каналов происходит от внешнего события или установкой разряда ADON регистра ADC_CR2.
  2. Если произошел запуск преобразования инжектированной группы при выполнении преобразования в регулярной группе, текущее преобразование сбрасывается, и выполняется преобразование инжектированной группы.
  3. Затем возобновляется преобразование регулярной группы с прерванного преобразования. Если во время запуска инжектированной группы произошло событие в регулярной группе, то его обработка откладывается до окончания преобразования инжектированной группы.

Использование DMA

Если в регулярной группе используется более одного канала, то для сохранения результата необходимо использовать DMA. Это позволяет избежать потери данных, которые уже хранятся в регистре ADC_DR. В конце преобразования регулярная группа формирует запрос DMA для сохранения результата из регистра ADC_DR в место, заданное пользователем.

Задание времени выборки

Время выборки для каждого канала задается разрядами SMP [2:0] регистров ADC_SMPR1 и ADC_SMPR2.

Для каждого канала время задается индивидуально.

Полное время преобразования вычисляется по формуле: Tconv = Sampling time + 12.5 cycles, где Sampling time – это заданное программистом время выборки.

Пример: Частота ADCCLK = 14 MHz, время выборки 1.5 цикла

Tconv = 1.5 + 12.5 = 14 циклов = 1μs.

Запуск преобразования от внешнего сигнала

Предусмотрен запуск преобразования от внешнего сигнала, которым может быть совпадение от таймеров, сигнал с внешней линии. Для использования внешних событий необходимо установить разряд EXTTRIG. Разряды EXTSEL[2:0] и JEXTSEL[2:0] задают источник запускающего сигнала для регулярной и инжектированной группы соответственно.

Таблица выбора источника запуска для регулярной и инжектированной группы:

Первые 6 источников – это сигнал от таймеров, седьмой – с внешней линии. А восьмой источник – это разряд регистра управления(для программного запуска)

Датчик температуры

Этот датчик используется для измерения температуры устройства.

Датчик подключен к 16 входу АЦП. Рекомендуемое время выборки при опросе датчика – 17.1 мкс

Схема подключения датчика температуры изображена ниже:

Когда датчик не используется, его можно отключить.

Напряжение на выходе датчика изменяется линейно с температурой.

В разных кристаллах эта линия смещается, что связано с технологией изготовления (до 45 градусов).

Внутренний датчик больше подходит для контроля изменения температуры, а не для измерения ее абсолютных значений (печально, но тоже не плохо).

Чтение температуры:

  1. Выбрать канал № 16 входным
  2. Задать время выборки 17.1 мкс
  3. Установить разряд TSVREFE для включения датчика (вывода из спящего режима)
  4. Запустить преобразование АЦП установкой разряда ADON (или внешним событием)
  5. Прочитать результат преобразования
  6. Вычислить значение температуры по формуле: Температура (в °C) = (V25 — VSENSE) / Avg_Slope + 25, где V25 – значение VSENSE при 25 градусах, VSENSE – измеренное значение, Avg_Slope – не понял.

Прерывания от АЦП

Прерывание может генерироваться по окончанию преобразования в регулярной или инжектированной группе, или при срабатывании оконного компаратора. Предусмотрено отдельное управление этими прерываниями.

В регистре ADC_SR есть еще два флага, но они не связаны с прерываниями:

  1. JSTRT – старт преобразования инжектированной группы
  2. STRT – старт преобразования в регулярной группе
