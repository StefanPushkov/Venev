STM32 TIMER Функция задержки

Старая-добрая функция delay для формирования временных задержек. С компиляторами для микроконтроллеров AVR обычно идут библиотеки с этой функцией. А вот в кейле для STM32 я такого не находил (может плохо искал?).

Но такую функцию можно соорудить самому используя таймеры.

Ведь, в конце концов, таймеры в первую очередь для этого предназначены – для отсчета времени.

Для этой цели задействуем таймер №2.

Настроим его, подав тактирование и обеспечив подачу на вход счетного регистра регистра таймера миллисекундных импульсов:

1
2
3

 RCC->APB1ENR |= RCC_APB1ENR_TIM2EN;   //подать тактирование на TIM2
 TIM2->PSC     = 8000-1;               //настроить делитель для формирования миллисекунд
 TIM2->CR1     = TIM_CR1_OPM;          //режим одного импульса

Во второй строке показана настройка входного делителя таймера.
8000 — это частота тактирования таймера в kHz.
Она будет зависить от настройки общей системы тактирования.

Дальше текст фукции, выполняющей задержку на заданное количество миллисекунд:

//*******************************************************************************************
//function  задержка времени                                                               //
//argument  длительность задержки от 1 до 65535 миллисекунд                                // 
//*******************************************************************************************
void delay_ms(uint16_t value)
{
  TIM2->ARR = value;                  //загрузить значение задержки
  TIM2->CNT = 0;
  TIM2->CR1 = TIM_CR1_CEN;	      //запустить таймер
  while((TIM2->SR & TIM_SR_UIF)==0){} //дождаться конца задержки
  TIM2->SR &= ~TIM_SR_UIF;	      //сбросить флаг
}

Фрагмент программы, использующей эту функцию:

1

 /*мигание светодиодом с частотой 1Hz                                                        */
 while(1)
 {
   delay_ms(500);         //задержка длительностью 500мс
   GPIOB->ODR ^= 0x0002;  //проинвертировать выход
 }

Функция не сложная, но весьма полезная.

      Похоже я ее не совсем коректно написал.
      Получается всегда на одну миллисекунду больше заданного.
      Если необходима задержка в 1 мс, то аргументом ф-ии передаем 1.
      В регистр ARR будет загружена единица.
      При запуске таймера счетный регистр начинает считать с нуля:
      1) При первом импульсе счетный регистр станет равен единице — это одна миллисекунда;
      2) При втором импульсе произойдет переполнение с установкой флага — это вторая миллисеекунда.
      Короче говоря, необходимо в фукции задержки уменьшать на 1 значение, загружаемое в регистр ARR:
      TIM2->ARR = value-1; //загрузить значение задержки
      Если этого не сделать, задержка всегда будет на 1 мс больше заданного.
      А как вы вычислили эту одну миллисекунду?
