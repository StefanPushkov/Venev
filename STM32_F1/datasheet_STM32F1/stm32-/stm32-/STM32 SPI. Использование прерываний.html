STM32 SPI. Использование прерываний

В модуле SPI предусмотрено 5 событий, могущих генерировать запрос на прерывание. Ниже приведена таблица из документации. В ней показано: события, генерирующие прерывания, соответствующие этим событиям  флаги и способ управления прерываниями от этих событий:

Первый столбец этой таблицы – события, генерирующие прерывания, второй – их флаги (расположены в регистре SR), третий – разряды регистра CR2, которые разрешают/запрещают прерывание от определенного события. Рассмотрим события подробнее:

  • Transmit buffer empty flag – буфер передатчика свободен;
  • Receive buffer not empty flag – в буфере приемника есть принятый байт;
  • Master Mode fault event – модуль был переведен из режима “Ведущий” в режим “Ведомый” по сигналу NSS;
  • Overrun error – потеряли принятый байт (не успели прочитать принятый байт, на его место принят другой);
  • CRC error flag – ошибка контрольной суммы;

Первые два события имеют собственные разряды, разрешающие прерывания от них – TXEIE и RXNEIE.

Три последних события объединены в одну группу – ошибка модуля. Прерывание от этой группы событий управляется одним разрядом  — ERRIE.

Разрешение прерываний от нужного события осуществляется следующим образом:

1
2
3

SPI1->CR2 |= SPI_CR2_TXEIE;        //разрешить прерывание по окончанию передачи               /
SPI1->CR2 |= SPI_CR2_RXNEIE;       //разрешить прерывание, если принят байт данных
SPI1->CR2 |= SPI_CR2_ERRIE;        //разрешить прерывание при возникновении ошибки


И не забываем разрешить прерывания от модуля SPI: 

1

NVIC_EnableIRQ (SPI1_IRQn);      //вызвать функцию, которая разрешит прерывание от модуля SPI1

//********************************************************************************************
//function  обработчик прерывания от SPI                                                    //
//********************************************************************************************
void SPI1_IRQHandler(void)
{
  uint16_t tmp;
 
  //причина прерывания - окончание передачи байта
  if(SPI1->SR &= SPI_SR_TXE)
  {
     //выполняем какие-то действия      
  }
 
  //причина прерывания - принят байт данных
  if(SPI1->SR &= SPI_SR_RXNE)
  {
     tmp = SPI1->DR;                //прочитать принятые данные
     //выполняем какие-то действия      
  }
 
  //причина прерывания - ошибка режима ведущего
  if(SPI1->SR &= SPI_SR_MODF)
  {
    //сбрасываем флаг события, которое вызвало прерывание и делаем что-то полезное
  }
 
  //причина прерывания - переполнение приемника данных
  if(SPI1->SR &= SPI_SR_OVR)
  {
    //сбрасываем флаг события, которое вызвало прерывание и делаем что-то полезное
  }
 
  //причина прерывания - ошибка данных
  if(SPI1->SR &= SSPI_SR_CRCERR)
  {
    //сбрасываем флаг события, которое вызвало прерывание и делаем что-то полезное
  }
 
}

Функция-обработчик определяет причину прерывания проверкой флагов. Обнаружив установленный флаг в некоторых случаях необходимо выполнить его сброс программно.

Пример использования прерывания.

Реализуем использование прерывания от SPI на практике. Задача – передать 512 байт из массива SPI_Buff[512]. Принимать ничего не будем, контролировать ошибки то же.

uint8_t  SPI_Buff[512];   //данные для передачи
uint16_t SpiCounter;      //показывает количество переданных байт
 
//********************************************************************************************
//function  обработчик прерывания от SPI                                                    //
//********************************************************************************************
void SPI1_IRQHandler(void)
{
  uint16_t tmp;
 
  //причина прерывания - окончание приема байта
  if(SPI1->SR &= SPI_SR_RXNE)
  {
     tmp = SPI1->DR;                  //прочитать принятый байт      
     SpiCounter++;                    //переместить указатель на следующий элемент массива
     if(SpiCounter<512)               //если конец массива не достигнут - передать байт
      SPI1->DR =  SPI_Buff[SpiCounter];      
  }
 
}

SPI1->DR = SPI_Buff[0];            //начать передачу с первого байта
while(SpiCounter<512);             //ждем передачу всего массива
SPI1->CR2 &= ~SPI_CR2_RXNEIE;      //запретить прерывание
